# DMM Webサービス API 利用規約 — コンプライアンスチェック

> 調査日: 2026-04-02
> 対象: Muse Log アプリケーション
> 参照: [DMM Webサービス利用規約](https://terms.dmm.com/affiliate_web_service/)

## 詳細

### 🔴 1. クレジット表記なし（リスク: **高**）

**問題点**
DMM Webサービス API の利用規約では、APIを通じて取得したコンテンツを表示する際に **`WEB SERVICE BY DMM.com` クレジットバッジの表示が義務付け** られています。現在の Muse Log には、このクレジット表記が実装されていません。

**根拠**
規約内に「WebサービスAPIを使用してコンテンツを掲載する場合は、DMM.comが提供するバナーを表示すること」と明示されています。

**推奨対応**
各画面のフッター（または女優情報・作品情報を表示している画面）に以下のバッジを追加する。

```html
<a href="https://affiliate.dmm.com/api/">
  <img
    src="https://pics.dmm.com/af/web_service/com_135_17.gif"
    width="135"
    height="17"
    alt="WEB SERVICE BY DMM.com"
  />
</a>
```

> ✅ 注: README.md の免責事項セクションにはすでに記載済みですが、**アプリ画面上での表示**が必要です。

---

### 🔴 2. 年齢確認なし（リスク: **高**）

**問題点**
Muse Log は成人向けコンテンツ（アダルト作品・女優情報）を扱うアプリですが、現在の実装ではサインアップ・ログイン時に **18歳以上であることの確認ステップが存在しません**。

**根拠**
DMM の API 利用規約および日本の法令（不正競争防止法・電気通信事業法等）では、成人向けコンテンツを提供するサービスに対して年齢確認措置が求められています。

**推奨対応**
- サインアップフロー（`/auth/signup`）に「18歳以上です」チェックボックスを追加
- または、ログイン前のトップページに年齢確認モーダルを表示する

---

### 🟡 3. 女優情報のDB保存（リスク: 中）

**問題点**
Muse Log では、DMM API から取得した女優情報（名前・画像URL・スペック等）を自社の PostgreSQL DB（`actresses` テーブル）にキャッシュとして保存しています。これが「APIから取得したデータの二次保存」に該当する可能性があります。

**現状の構成**

```
DMM API → Go Backend → Supabase DB (actresses テーブル)
```

**グレーゾーンの理由**
一般に「データの再頒布」は禁止されていますが、パフォーマンス目的のキャッシュについては規約の解釈が曖昧です。個人利用・非商用の範囲であれば許容されるケースもありますが、明示的な許可はありません。

**推奨対応**
- DMM サポートに利用目的（個人利用・非商用）を伝えた上で、DB保存の可否を問い合わせる
- または、DB 保存をやめて毎回 DMM API を直接呼び出す構成に変更する（ただしレート制限に注意）

---

### 🟡 4. 非アフィリエイト利用（リスク: 中）

**問題点**
DMM Webサービス API は本来 **アフィリエイト目的**（商品リンクを貼って収益を得る）のために提供されています。Muse Log は収益化を目的としていないため、API の本来の利用目的と乖離している可能性があります。

**グレーゾーンの理由**
個人の技術研究・非商用であれば問題ないケースも多いですが、規約上は「アフィリエイト目的での利用」が前提となっています。README.md の免責事項に「個人の技術研究を目的とした非公式アプリ」と記載されている点は一定の配慮ですが、規約上のグレーゾーンには変わりありません。

**推奨対応**
- アフィリエイトリンク（`fanza_url`）を各女優カードに表示することで、アフィリエイト利用の体裁を整える（`actresses` テーブルには `fanza_url` カラムが存在する）
- DMM に利用目的を申告・確認する

---

## 対応ロードマップ（推奨）

```
Phase 1 (即時対応)
├── クレジットバッジを全画面フッターに追加
└── サインアップ時に年齢確認チェックボックスを追加

Phase 2 (短期対応)
├── DMM サポートへ利用目的の確認メールを送付
└── 女優カードにアフィリエイトリンク（fanza_url）を表示

Phase 3 (中長期)
└── DB保存方針をDMM の回答に基づき再検討
```
