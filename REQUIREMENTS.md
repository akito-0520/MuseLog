## 1. 全体共通要素

### 1.1 ヘッダー

#### 概要

全画面で表示される共通ヘッダー。ロゴとナビゲーション機能を提供。

#### 実装する機能

- **ロゴ表示**: クリックで一覧画面（ホーム）に遷移
- **ユーザーアイコン**: ログイン状態を表示
  - 未ログイン時: "ログイン"ボタン
  - ログイン時: アイコン（クリックでドロップダウンメニュー）
- **検索ボタン**: グローバル検索画面へ遷移（モバイルは固定表示）

#### 技術要件

- Next.js Layout で実装
- ログイン状態は Supabase Auth の `useUser()` フックで管理

---

### 1.2 フッター（ボトムナビゲーション）

#### 概要

モバイルのみ表示される固定フッター。主要画面への導線。

#### 実装する機能

- **一覧**: お気に入り一覧画面（ホーム）
- **検索**: 女優検索画面
- **設定**: 設定画面

#### UI要件

- **デザイン**: アイコン + ラベル、アクティブ状態を色で表現

#### 技術要件

- Next.js の `usePathname()` でアクティブ状態を判定
- Lucide React でアイコン表示

---

### 1.3 ローディング状態

#### 実装パターン

**スケルトンローダー**

- 使用箇所: 一覧画面、検索結果、詳細画面
- デザイン: カード形状を模したプレースホルダー

#### 技術要件

- React Suspense で実装
- `loading.tsx` ファイルで各ルートのローディングUIを定義

---

### 1.4 エラー表示

#### エラーパターンと表示方法

| エラータイプ             | 表示方法       | 表示内容                                                         | 例                                       |
| :----------------------- | :------------- | :--------------------------------------------------------------- | :--------------------------------------- |
| **ネットワークエラー**   | トースト通知   | "接続エラーが発生しました。再度お試しください。"                 | API接続失敗                              |
| **認証エラー**           | モーダル       | "セッションが切れました。再ログインしてください。"               | JWTトークン期限切れ                      |
| **バリデーションエラー** | インライン表示 | フォーム下に赤文字で表示                                         | "メールアドレスの形式が正しくありません" |
| **404エラー**            | 専用ページ     | "ページが見つかりません" + ホームへのリンク                      | 存在しないルートにアクセス               |
| **500エラー**            | 専用ページ     | "サーバーエラーが発生しました" + リロードボタン                  | バックエンドエラー                       |
| **DMM APIエラー**        | トースト通知   | "検索結果の取得に失敗しました。時間をおいて再度お試しください。" | レート制限、API障害                      |

#### 技術要件

- トースト通知: `sonner` ライブラリを使用
- エラーバウンダリ: Next.js `error.tsx` で実装

## 2. 認証関連画面

### 2.1 新規登録画面（Sign Up）

#### 概要

新規ユーザーのアカウント作成。OAuth（Google）とメール/パスワードの2方式に対応。

#### 実装する機能

- [ ] **OAuth (Google) ログイン**（MVP）
  - Googleアカウントでワンクリック登録
  - Supabase Auth の `signInWithOAuth` を使用

- [ ] **メール/パスワード登録**
  - メールアドレス、パスワード（8文字以上）を入力
  - 確認メール送信（メールアドレス認証）
  - Supabase Auth の `signUp` を使用

#### UI要件

- **レイアウト**: 中央配置、カード形式
- **フォーム要素**:
  - メールアドレス入力（type="email"）
  - パスワード入力（type="password"、表示/非表示切り替えボタン）
  - パスワード確認入力
  - 「登録」ボタン
  - 「Googleで登録」ボタン（Googleロゴ付き）
- **バリデーション**:
  - メールアドレス形式チェック（正規表現）
  - パスワード長チェック（8文字以上）
  - パスワード一致チェック
  - リアルタイムバリデーション（フォーカスアウト時）

#### フロー

1. ユーザーが情報を入力
2. バリデーション実行
3. Supabase Auth にリクエスト送信
4. 成功: 確認メール送信 → "確認メールを送信しました" 画面へ遷移
5. 失敗: エラーメッセージ表示（"このメールアドレスは既に登録されています" 等）

#### 技術要件

- **フォームライブラリ**: `react-hook-form` + `zod` (バリデーション)
- **Supabase Auth**: `supabase.auth.signUp()` / `supabase.auth.signInWithOAuth()`
- **遷移先**: ログイン後は `/actresses`（一覧画面）へリダイレクト

#### セキュリティ要件

- CSRF対策: Supabase Auth が自動対応
- パスワード要件: 最低8文字、英数字混在推奨
- reCAPTCHA: Phase 3 で検討（スパム登録防止）

---

### 2.2 ログイン画面（Login）

#### 概要

既存ユーザーのログイン。

#### 実装する機能（Phase 1: MVP）

- [ ] **OAuth (Google) ログイン**

#### 実装する機能（Phase 3: 追加機能）

- [ ] **メール/パスワード ログイン**
  - Supabase Auth の `signInWithPassword` を使用
- [ ] **「パスワードを忘れた」リンク**
  - パスワードリセット画面へ遷移

#### UI要件

- 新規登録画面と同様のレイアウト
- 「ログイン」ボタン、「Googleでログイン」ボタン
- 「アカウントをお持ちでない方」→ 新規登録画面へのリンク

#### フロー

1. ユーザーがメール/パスワードを入力（またはGoogle認証）
2. Supabase Auth にリクエスト送信
3. 成功: JWTトークン取得 → `/actresses` へリダイレクト
4. 失敗: "メールアドレスまたはパスワードが正しくありません" エラー表示

#### 技術要件

- **セッション管理**: Supabase Auth が自動で Cookie に保存
- **リダイレクト**: ログイン前のページに戻る機能（`redirect` パラメータ）

---

### 2.3 パスワードリセット画面（Phase 3）

#### 実装する機能

- [ ] メールアドレス入力でリセットリンクを送信
- [ ] リセットリンクから新パスワード設定画面へ遷移
- [ ] 新パスワード入力・更新

#### フロー

1. ログイン画面から「パスワードを忘れた」をクリック
2. メールアドレス入力 → `supabase.auth.resetPasswordForEmail()`
3. メールに記載されたリンクをクリック
4. 新パスワード入力画面 → `supabase.auth.updateUser({ password: newPassword })`
5. 完了 → ログイン画面へ遷移

---

### 2.4 プロフィール編集画面（Phase 2）

#### 実装する機能

- [ ] ニックネーム変更
- [ ] メールアドレス変更（再認証が必要）
- [ ] パスワード変更（現在のパスワード入力が必要）
- [ ] アカウント削除（確認モーダル表示）

#### UI要件

- 設定画面からアクセス
- フォーム形式で各項目を編集
- 「保存」ボタンでバックエンドに送信

#### フロー（メールアドレス変更）

1. 新しいメールアドレスを入力
2. `supabase.auth.updateUser({ email: newEmail })` 実行
3. 新旧両方のメールアドレスに確認メール送信
4. リンククリックで変更完了

## 3. 一覧・詳細表示画面

### 3.1 お気に入り一覧画面（Home）

#### 概要

ログインユーザーがお気に入り登録した女優を一覧表示。アプリのメイン画面。

#### 実装する機能

- **カード形式の一覧表示**
  - 女優名、サムネイル画像、星評価、タグを表示
  - グリッドレイアウト（デスクトップ: 4列、タブレット: 3列、モバイル: 2列）
- **ページネーション**
  - 1ページ20件表示
  - ページ番号クリックで遷移
  - 無限スクロールは Phase 2 で検討
- **ソート機能**
  - 登録日順（デフォルト）
  - 評価順（高評価順/低評価順）
  - 名前順（あいうえお順）
- **フィルター機能**
  - タグでフィルタ（複数選択可）
  - 評価でフィルタ（3以上、4以上等）
- **検索バー**
  - 女優名で部分一致検索
  - リアルタイム検索（入力中に結果を更新）

#### UI要件

- **カードデザイン**:
  - 縦長カード（アスペクト比 2:3）
  - ホバー時に拡大アニメーション
  - クリックで詳細モーダル表示
- **空状態**:
  - お気に入りが0件の場合: "まだお気に入りがありません" + 検索画面へのリンク表示
- **画像読み込みエラー**:
  - 画像が取得できない場合: プレースホルダー画像表示（グレーの人物アイコン）

#### データ取得

```
GET /api/reviews?page=1&sort=created_at&order=desc&tag=巨乳&rating_min=4
```

#### レスポンス例

```json
{
  "reviews": [
    {
      "id": 123,
      "actress": {
        "id": 456,
        "name": "山田花子",
        "image_url": "https://...",
        "fanza_url": "https://..."
      },
      "rating": 5,
      "tags": ["巨乳", "美人"],
      "created_at": "2024-03-01T10:00:00Z"
    }
  ],
  "total": 100,
  "page": 1,
  "per_page": 20
}
```

#### 技術要件

- **データフェッチ**: Next.js Server Components で初回データ取得
- **クライアント操作**: React Query でページネーション・ソート・フィルターを管理
- **画像最適化**: Next.js `<Image>` コンポーネント使用、lazy loading 有効
- **Suspense**: `loading.tsx` でスケルトンローダー表示

#### パフォーマンス要件

- 初回表示: 2秒以内
- ページ遷移: 500ms以内（キャッシュ利用）

## 4. 検索・インポート画面

### 4.1 女優検索画面

#### 概要

DMM API を使用して女優を検索し、お気に入りに追加。

#### 実装する機能

- **検索バー**
  - 女優名で検索（部分一致）
  - 曖昧検索対応（"やまだ" → "山田" も検索）
- **検索結果表示**
  - カード形式（一覧画面と同様）
  - 「追加」ボタン表示（既にお気に入りの場合は「追加済み」を表示）
- **ページネーション**
  - 1ページ20件表示
  - DMM API の結果をそのまま表示
- **重複チェック**
  - 既にお気に入り登録済みの女優には「追加済み」バッジ表示
  - 追加ボタンを無効化

#### フロー

1. ユーザーが検索キーワードを入力
2. フロントエンドがバックエンド API を呼び出し
3. バックエンドが DMM API にリクエスト送信
4. 結果をフロントエンドに返却・表示
5. ユーザーが「追加」ボタンをクリック
6. 評価モーダル表示（オプション: スキップ可能）
7. お気に入り登録完了 → トースト通知表示

#### データフロー

```
Frontend → Backend API → DMM API
         ← (Cache)    ←

Frontend: 検索キーワード送信
Backend: DMM API 呼び出し、Redis にキャッシュ（TTL: 24時間）
Frontend: 結果表示
```

#### API仕様

```
GET /api/search/actresses?q=山田&page=1
```

レスポンス:

```json
{
  "actresses": [
    {
      "dmm_actress_id": "12345",
      "name": "山田花子",
      "image_url": "https://...",
      "fanza_url": "https://...",
      "bust": 88,
      "waist": 58,
      "hip": 86,
      "height": 160,
      "cup": "E",
      "is_favorited": false
    }
  ],
  "total": 50,
  "page": 1
}
```

#### UI要件

- **空状態**: 検索結果が0件の場合 → "該当する女優が見つかりませんでした"
- **ローディング**: 検索中はスピナー表示
- **エラー**: DMM API エラー時 → "検索に失敗しました。時間をおいて再度お試しください。"

#### 技術要件

- **デバウンス**: 検索入力に300msのデバウンス適用（連続入力時のAPI呼び出し削減）
- **キャッシュ**: Redis で DMM API レスポンスをキャッシュ（TTL: 24時間）
- **レート制限**: 1ユーザーあたり 10リクエスト/分（バックエンドで制御）

### 4.2 評価モーダル（お気に入り追加時）

#### 概要

お気に入り追加時に評価を入力できるモーダル。スキップも可能。

#### 実装する機能

- **星評価入力** (1〜5、オプション)
- **お気に入り動画タイトル入力** (オプション)
- **お気に入り動画URL入力** (オプション)
- **メモ入力** (オプション、最大500文字)
- **タグ選択** (既存タグから選択 or 新規作成)
- **「保存」ボタン**: DB に保存
- **「スキップ」ボタン**: 評価なしでお気に入り登録

#### フロー

1. 検索画面で「追加」ボタンをクリック
2. 評価モーダル表示
3. ユーザーが情報を入力（またはスキップ）
4. 「保存」 → バックエンド API にPOST送信
5. 成功 → モーダルを閉じる、トースト通知表示
6. 失敗 → エラーメッセージ表示

#### API仕様

```
POST /api/reviews
Body:
{
  "actress_id": 456,
  "rating": 5,
  "favorite_video_title": "タイトル",
  "favorite_video_url": "https://...",
  "memo": "メモ",
  "tag_ids": [1, 2, 3]
}
```

#### 技術要件

- **フォームバリデーション**: `zod` で URL形式チェック、文字数制限
- **タグ入力**: Combobox UI（選択 + 新規作成）

## 7. 共有機能

### 7.1 OGP画像生成・共有（Phase 2）

#### 概要

お気に入りリストを美しいOGP画像として生成し、SNSで共有できる機能。

#### 実装する機能

- **共有ボタン**: 一覧画面に配置
- **OGP画像生成**: バックエンドで動的生成（Go + 画像ライブラリ）
- **共有URL発行**: `/share/:share_id` 形式の公開URL
- **SNS共有ボタン**: Twitter, Facebook 等

#### フロー

1. ユーザーが「共有」ボタンをクリック
2. バックエンドがOGP画像を生成（女優画像 + 星評価 + タグを合成）
3. Supabase Storage に画像アップロード
4. 共有用URLを発行（`https://muselog.app/share/abc123`）
5. SNS共有ボタン表示

#### OGP画像仕様

- サイズ: 1200x630px
- 内容: 最大9枚の女優画像をグリッド配置、タイトル、ユーザー名

#### 技術要件

- **画像生成**: Go の `imaging` ライブラリ使用
- **ストレージ**: Supabase Storage（公開バケット）
- **キャッシュ**: 同じリストの画像は再生成しない（TTL: 7日間）

## 📊 非機能要件

### パフォーマンス目標

- **一覧画面**: 初回表示 2秒以内、2回目以降 1秒以内
- **検索**: 入力から結果表示まで 1秒以内
- **詳細画面**: 500ms以内

### アクセシビリティ目標

- **WCAG 2.1 AA 準拠**
