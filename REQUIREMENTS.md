## 1. 全体共通要素

### 1.1 ヘッダー

#### 概要

全画面で表示される共通ヘッダー。ロゴのみを配置したシンプルな構成。

#### 実装する機能

- **ロゴ表示**: クリックで一覧画面（`/actresses`）に遷移
  - テキスト: `MuseLog💋`、ホバー時にティールカラー (`#00d4d4`) に変化

#### UI要件

- **高さ**: `h-14`（56px）、`fixed` 配置（`z-50`）
- **背景**: `#0a1a1a` / 95% 不透明度 + backdrop-blur
- **ボーダー**: 下部に `border-[#1e3333]`

#### 技術要件

- Next.js Layout (`app/(main)/layout.tsx`) で実装
- ユーザーアイコン・ログインボタン・グローバル検索ボタンは**ヘッダーに配置しない**

---

### 1.2 フッター（ボトムナビゲーション）

#### 概要

全デバイスで表示される固定フッター。主要3画面への導線。

#### ナビゲーション項目（左から順）

| アイコン   | ラベル | 遷移先       |
| :--------- | :----- | :----------- |
| Search     | 検索   | `/search`    |
| LayoutGrid | 一覧   | `/actresses` |
| Settings   | 設定   | `/settings`  |

#### UI要件

- **高さ**: `h-16`（64px）、`fixed` 配置（`z-40`）
- **背景**: `#0d1f1f`、上部ボーダー `border-[#1e3333]`
- **アイコンサイズ**: 22px（Lucide React）
- **アクティブ状態**: テキスト・アイコンが `#00d4d4` に変化（未アクティブ: `gray-500`）
- **レイアウト**: `justify-around` で均等配置

#### 技術要件

- Next.js の `usePathname()` でアクティブ状態を判定
- アイコンは Lucide React (`Search`, `LayoutGrid`, `Settings`)

---

### 1.3 ローディング状態

#### 実装パターン

**スケルトンローダー**

- 使用箇所: 一覧画面（カード形式）、検索結果（リスト行形式）
- デザイン: 表示する要素の形状に合わせたプレースホルダー
- アニメーション: `animate-pulse`
- カラー: `bg-[#1a2e2e]`

#### 技術要件

- 各ページコンポーネント内で `isLoading` 状態により切り替え
- `actresses/loading.tsx` でページレベルのローディング UI を定義

---

### 1.4 エラー表示

#### エラーパターンと表示方法

| エラータイプ             | 表示方法       | 表示内容                                             | 例                                       |
| :----------------------- | :------------- | :--------------------------------------------------- | :--------------------------------------- |
| **ネットワークエラー**   | トースト通知   | "接続エラーが発生しました。再度お試しください。"     | API接続失敗                              |
| **認証エラー**           | モーダル       | "セッションが切れました。再ログインしてください。"   | JWTトークン期限切れ                      |
| **バリデーションエラー** | インライン表示 | フォーム下に赤文字 (`text-red-400`) で表示           | "メールアドレスの形式が正しくありません" |
| **404エラー**            | 専用ページ     | "ページが見つかりません" + ホームへのリンク          | 存在しないルートにアクセス               |
| **500エラー**            | 専用ページ     | "サーバーエラーが発生しました" + リロードボタン      | バックエンドエラー                       |
| **データ取得失敗**       | インライン表示 | `text-red-400` のエラーメッセージ + 再読み込みリンク | `isError` 時                             |

#### 技術要件

- トースト通知: 検索画面は自前実装の `Toast` コンポーネント（`sonner` は将来検討）
- エラーバウンダリ: Next.js `error.tsx` で実装

## 2. 認証関連画面

### 2.1 新規登録画面（Sign Up）

#### 概要

新規ユーザーのアカウント作成。OAuth（Google・Apple）とメール/パスワードの方式に対応。

#### 実装する機能

- [ ] **OAuth (Google) ログイン**（MVP）
  - Supabase Auth の `signInWithOAuth` を使用

- [ ] **OAuth (Apple) ログイン**
  - Supabase Auth の `signInWithOAuth` を使用

- [ ] **メール/パスワード登録**
  - メールアドレス、パスワード（8文字以上）を入力
  - 確認メール送信（メールアドレス認証）
  - Supabase Auth の `signUp` を使用

#### UI要件

- **レイアウト**: 中央配置、カード形式
- **フォーム要素**:
  - メールアドレス入力（type="email"）
  - パスワード入力（type="password"、表示/非表示切り替えボタン）
  - パスワード確認入力
  - 「登録」ボタン
  - 「Googleで登録」ボタン（Googleロゴ付き）
  - 「Appleで登録」ボタン（Appleロゴ付き）
- **バリデーション**:
  - メールアドレス形式チェック（正規表現）
  - パスワード長チェック（8文字以上）
  - パスワード一致チェック
  - リアルタイムバリデーション（フォーカスアウト時）

#### フロー

1. ユーザーが情報を入力
2. バリデーション実行
3. Supabase Auth にリクエスト送信
4. 成功: 確認メール送信 → "確認メールを送信しました" 画面へ遷移
5. 失敗: エラーメッセージ表示（"このメールアドレスは既に登録されています" 等）

#### 技術要件

- **フォームライブラリ**: `react-hook-form` + `zod` (バリデーション)
- **Supabase Auth**: `supabase.auth.signUp()` / `supabase.auth.signInWithOAuth()`
- **遷移先**: ログイン後は `/actresses`（一覧画面）へリダイレクト

---

### 2.2 ログイン画面（Login）

#### 概要

既存ユーザーのログイン。

#### 実装する機能

- [ ] **OAuth (Google) ログイン**
- [ ] **OAuth (Apple) ログイン**
- [ ] **メール/パスワード ログイン**
  - Supabase Auth の `signInWithPassword` を使用
- [ ] **「パスワードを忘れた」リンク**
  - パスワードリセット画面へ遷移

#### UI要件

- 新規登録画面と同様のレイアウト
- 「ログイン」ボタン、「Googleでログイン」ボタン、「Appleでログイン」ボタン
- 「アカウントをお持ちでない方」→ 新規登録画面へのリンク

#### フロー

1. ユーザーがメール/パスワードを入力（またはOAuth認証）
2. Supabase Auth にリクエスト送信
3. 成功: JWTトークン取得 → `/actresses` へリダイレクト
4. 失敗: "メールアドレスまたはパスワードが正しくありません" エラー表示

#### 技術要件

- **セッション管理**: Supabase Auth が自動で Cookie に保存
- **リダイレクト**: ログイン前のページに戻る機能（`redirect` パラメータ）

---

### 2.3 パスワードリセット画面

#### 実装する機能

- [ ] メールアドレス入力でリセットリンクを送信
- [ ] リセットリンクから新パスワード設定画面へ遷移
- [ ] 新パスワード入力・更新

#### フロー

1. ログイン画面から「パスワードを忘れた」をクリック
2. メールアドレス入力 → `supabase.auth.resetPasswordForEmail()`
3. メールに記載されたリンクをクリック
4. 新パスワード入力画面 → `supabase.auth.updateUser({ password: newPassword })`
5. 完了 → ログイン画面へ遷移

## 3. 一覧・詳細表示画面

### 3.1 お気に入り一覧画面（`/actresses`）

#### 概要

ログインユーザーがお気に入り登録した女優を一覧表示。アプリのメイン画面。

#### 実装する機能

- **カード形式の一覧表示**
  - 女優名、サムネイル画像、星評価、最終更新日を表示
  - グリッドレイアウト（デフォルト: 2列、sm: 3列、lg: 4列）
- **ページネーション**
  - 1ページ20件表示
  - 前へ/次へボタン + ページ番号ボタン
- **ソート機能**（ドロップダウンメニュー）
  - 登録日順（デフォルト）
  - 評価順（高い順）
  - 評価順（低い順）
  - 名前順（あいうえお）
- **検索バー**
  - 女優名で部分一致検索（スペース除去して比較）
  - 入力のたびに結果を更新（クライアントサイドフィルタリング）
- **FABボタン**（Floating Action Button）
  - 画面右下（ボトムナビ上）に固定表示
  - `+` アイコン、クリックで女優追加フロー（TODO: 検索ページまたはモーダルへ）

#### UI要件

- **カードデザイン**:
  - アスペクト比 2:3 の縦長カード
  - ホバー時に画像が `scale-105` で拡大
  - クリックで詳細モーダル表示
  - 画像下に女優名・星評価・最終更新日を表示
- **空状態**:
  - お気に入りが0件の場合: `User` アイコン + "まだお気に入りがありません" + 検索画面へのリンク
- **画像なし状態**:
  - `image_url` が null の場合: グレーの `User` アイコン（48px）を表示
- **エラー状態**:
  - `isError` 時: エラーメッセージ + 「再読み込み」リンク

#### データ取得

```
GET /api/reviews?page=1&sort=created_at&order=desc&q=検索ワード
```

#### レスポンス例

```json
{
  "reviews": [
    {
      "id": 123,
      "actress": {
        "id": 456,
        "name": "山田花子",
        "image_url": null,
        "fanza_url": "https://...",
        "height": 163,
        "bust": 88,
        "waist": 58,
        "hips": 86,
        "cup": "E"
      },
      "rating": 5.0,
      "tags": ["巨乳", "美人"],
      "memo": "メモ",
      "favorite_videos": [{ "title": "タイトル", "url": "https://..." }],
      "created_at": "2024-03-01T10:00:00Z",
      "updated_at": "2024-03-01T10:00:00Z"
    }
  ],
  "total": 100,
  "page": 1,
  "per_page": 20
}
```

#### 技術要件

- **データフェッチ**: `"use client"` + TanStack Query（`useQuery`）で管理
- **ソート/検索**: `queryKey` に `{ page, sort, q }` を含め、変更時に再フェッチ
- **`keepPreviousData`**: ページ遷移時に前データを維持（`placeholderData: keepPreviousData`）
- **画像最適化**: Next.js `<Image>` コンポーネント使用、`fill` + `object-cover`

### 3.2 詳細モーダル（`ActressDetailModal`）

#### 概要

一覧画面のカードをクリックすると、詳細情報をモーダルで表示。

#### 実装する機能

- **女優情報表示**
  - 横長レイアウト: 左に画像（w-32、2:3比）、右に名前・星評価・スペック

- **削除ボタン**
  - クリックで `DeleteConfirmDialog` を表示 → 確認後に削除 + モーダルを閉じる
- **編集ボタン**
  - `/actresses/[actress.id]` へ遷移（`<a>` タグ）
- **FANZAリンク**: `fanza_url` がある場合のみフッターに表示

#### UI要件

- **モーダルサイズ**: 最大幅 600px、最大高さ 90dvh
- **背景**: `bg-[#0d1f1f]`、ボーダー `border-[#1e3333]`、`rounded-2xl`

- **フッター**: 固定、FANZAリンク（左）+ 削除ボタン + 編集ボタン（右）
- **実装**: shadcn/ui `Dialog` コンポーネント使用

#### 技術要件

- **ライブラリ**: `@/components/ui/dialog`（shadcn/ui）
- **データ取得**: 一覧で既に取得済みのデータを props で渡す（追加APIコール不要）

## 4. 検索・インポート画面

### 4.1 女優検索画面（`/search`）

#### 概要

DMM API を使用して女優を検索し、お気に入りに追加。

#### 実装する機能

- **検索バー**
  - 女優名で検索（部分一致・ルビ読み仮名も対象）
  - 入力クリアボタン（`×` アイコン）
  - 検索結果件数表示 + 「条件をクリア」ボタン
- **検索結果表示**
  - リスト行形式（`ActressRow`）で表示
  - サムネイル（w-16 h-16、rounded-xl）、女優名 + ルビ、スペック（身長・バスト・ウエスト）
  - 「追加」ボタン（`+` アイコン付き丸ボタン）または「登録済」表示
- **ページネーション**
  - 1ページ20件表示
  - 前へ/次へ + ページ番号ボタン
- **重複チェック**
  - 追加済みの女優には `Check` アイコン + 「登録済」ラベルを表示
  - 追加ボタンを表示しない（クライアント側の `favorites` Set で管理）

#### フロー

1. ユーザーが検索キーワードを入力（400ms デバウンス）
2. フロントエンドがバックエンド API を呼び出し
3. バックエンドが DMM API にリクエスト送信
4. 結果をリスト行形式で表示
5. ユーザーが「追加」ボタンをクリック
6. `AddReviewModal` を表示
7. 保存 → トースト通知 + `favorites` Set に追加

#### API仕様

```
GET /api/dmm/actresses?keyword=山田&page=1&per_page=20
```

レスポンス:

```json
{
  "actresses": [
    {
      "id": "12345",
      "name": "山田花子",
      "ruby": "やまだ はなこ",
      "image_url": null,
      "fanza_url": "https://...",
      "height": 160,
      "bust": 88,
      "waist": 58,
      "hips": 86,
      "cup": "E"
    }
  ],
  "total": 50,
  "page": 1,
  "per_page": 20
}
```

#### UI要件

- **プレースホルダー**: "女優名で検索..."（`autoFocus` あり）
- **空状態（検索前）**: 何も表示しない
- **空状態（0件）**: `User` アイコン + "「{keyword}」に一致する女優が見つかりませんでした"
- **ローディング**: スケルトン行（`RowSkeleton`）を4件表示
- **ページ切り替え中**: 中央にスピナー表示（`isFetching && !isLoading`）
- **トースト**: 追加成功時に "{name} をお気に入りに追加しました"（3秒で自動消去）

#### 技術要件

- **デバウンス**: 400ms（`useRef` + `setTimeout` で実装）
- **クエリ有効化**: `keyword.trim().length > 0` の場合のみ `enabled: true`
- **レート制限**: 1ユーザーあたり 10リクエスト/分（バックエンドで制御）

### 4.2 評価モーダル（`AddReviewModal`）

#### 概要

お気に入り追加時に評価・タグ・メモ・動画を入力できるモーダル。

#### 実装する機能

- **女優情報表示（読み取り専用）**
  - 画像（w-32、2:3比）、女優名、ルビ、スペック（身長・スリーサイズ・カップ）
  - FANZAリンク（`fanza_url` がある場合）
- **星評価入力** (`StarRatingInput`): 0〜5（0 = 評価なし、`allowZero` 対応）
- **タグ選択** (`TagSelector`): 既存タグから選択または新規作成（最大5個）
- **メモ入力**: テキストエリア（最大 `MAX_MEMO` 文字、文字数カウンター付き）
- **お気に入り動画**: DMM APIから取得したURLを複数登録可能（タイトル + URL の組み合わせ）
  - 「動画を追加」ボタン、個別削除ボタン
- **「キャンセル」ボタン**: モーダルを閉じる（追加なし）
- **「追加する」ボタン**: データを保存してモーダルを閉じる

#### フロー

1. 検索画面で「追加」ボタンをクリック
2. `AddReviewModal` 表示
3. ユーザーが情報を入力（または何も入力せずに「追加する」）
4. 「追加する」 → バックエンド API にPOST送信
5. 成功 → モーダルを閉じる、トースト通知表示
6. 「キャンセル」 → モーダルを閉じる（登録なし）

#### API仕様

```
POST /api/reviews
Body:
{
  "actress_id": "12345",
  "rating": 5,
  "tags": ["巨乳", "美人"],
  "memo": "メモ"
}
```

#### 技術要件

- **タグ**: `string[]`（タグ名の配列、IDではない）
- **お気に入り動画**: URLはDMM APIから取得するためDBへの保存は不要
- **フォームリセット**: 「追加する」後にローカル状態をリセット

## 5. 女優詳細・編集画面

### 5.1 概要

女優の詳細情報を表示し、評価・タグ・メモ・動画を編集できる専用ページ（`/actresses/[id]`）。

### 5.2 実装する機能

- **女優情報表示（読み取り専用）**
  - 画像（2:3比）、女優名、身長、スリーサイズ、カップ（`SpecRow` コンポーネント）
  - `fanza_url` がある場合はフッターにFANZAリンクを表示
- **評価編集フォーム**（`react-hook-form` + `zod`）
  - **評価**: `StarRatingInput` コンポーネント（1〜5）
  - **タグ**: `TagSelector` コンポーネント（最大5個）
  - **メモ**: テキストエリア（最大 `MAX_MEMO` 文字、文字数カウンター付き）
  - **お気に入り動画**: DMM APIから取得したURLを `useFieldArray` で複数管理（タイトル + URL）
    - 「動画を追加」ボタン（破線ボーダー）、個別削除ボタン
- **「更新する」ボタン**: フォームに変更がある場合のみ有効（`isDirty`）
- **未保存インジケータ**: `isDirty` 時にヘッダーに「未保存」テキスト表示
- **お気に入りから削除**: 「お気に入りから削除」ボタン → `DeleteConfirmDialog` → 削除後 `/actresses` へ遷移

### 5.3 UI要件

- **レイアウト**:
  - モバイル: 縦並び（画像カード → フォームセクション群）
  - デスクトップ（lg以上）: 2カラム（左: 女優情報カード 260px・sticky、右: フォームセクション群）
- **フォームセクション**: 各項目をカード（`rounded-2xl bg-[#0d1f1f] border border-[#1e3333]`）で囲む
- **固定フッター**: ボトムナビの上（`bottom-16`）に固定
  - 左: FANZAリンク（または「公式ページ未設定」）
  - 右: 「更新する」ボタン（送信中はスピナー、成功時は `Check` アイコン + 「保存済み」2.5秒表示）
- **削除エリア**: フォーム下部に赤色のアウトラインボタン

### 5.4 データ取得

```
GET /api/reviews/:id
```

レスポンス:

```json
{
  "id": 123,
  "actress": {
    "id": 456,
    "name": "山田花子",
    "image_url": null,
    "fanza_url": "https://...",
    "height": 163,
    "bust": 88,
    "waist": 58,
    "hips": 86,
    "cup": "E"
```

  - ユーザーアイコン（デフォルト: `User` アイコン）

- **連携アカウント（OAuthプロバイダー）**
  - Google・Apple の連携/解除ができる `OAuthRow` を表示
  - 連携済みの場合: 「連携済み」バッジ + 「解除」ボタン
  - 未連携の場合: 「連携する」ボタン
  - 解除時: `DeleteConfirmDialog` で確認
- **表示設定**
  - 1ページの表示件数選択（10件 / 20件 / 30件 / 50件、`<select>` で変更）
- **ログアウト**
  - `DeleteConfirmDialog` で確認 → `supabase.auth.signOut()` → `/login` へ遷移
- **アカウント削除**（`DeleteAccountModal`）
  - 確認 → 全データ削除 → ログアウト

## 7. 共有機能

### 7.1 OGP画像生成・共有（検討中）

#### 概要

お気に入りリストを美しいOGP画像として生成し、SNSで共有できる機能。

#### 実装する機能

- **共有ボタン**: 一覧画面に配置
- **OGP画像生成**: バックエンドで動的生成（Go + 画像ライブラリ）
- **共有URL発行**: `/share/:share_id` 形式の公開URL
- **SNS共有ボタン**: Twitter, Facebook 等

#### フロー

1. ユーザーが「共有」ボタンをクリック
2. バックエンドがOGP画像を生成（女優画像 + 星評価 + タグを合成）
3. Supabase Storage に画像アップロード
4. 共有用URLを発行（`https://muselog.app/share/abc123`）
5. SNS共有ボタン表示

#### OGP画像仕様

- サイズ: 1200x630px
- 内容: 最大9枚の女優画像をグリッド配置、タイトル、ユーザー名

#### 技術要件

- **画像生成**: Go の `imaging` ライブラリ使用
- **ストレージ**: Supabase Storage（公開バケット）
- **キャッシュ**: 同じリストの画像は再生成しない（TTL: 7日間）
